### MAY (informational, accepted)

#### ADR-0003 — two unary relay helpers omit a leading `ctx` parameter (stylistic)

- **Rule (MAY):** thread `context.Context` as the first parameter through
  request-path functions, for consistency.
- **File:** `internal/router/proxy.go` (`handleUnaryResponse`,
  `handleNativeUnaryResponse`).
- **Note:** these leaf helpers take `(resp, sink, outcome)` with no `ctx`, unlike
  their streaming siblings. This breaks no MUST: `ctx` already crosses every layer
  boundary and is bound into the `be.Chat(ctx, …)` / `be.ChatNative(ctx, …)` call
  whose `resp.Body` they read, so cancellation still propagates and no goroutine
  is spawned. Accepted as-is; adding `ctx` for symmetry is optional.

## What Was Verified

This audit combined mechanical checks with targeted code reading against a clean
green gate (`go build ./...`, `go vet ./...`, `gofmt -l .` empty, and
`go test -race ./...` all pass; `go vet -tags integration ./...` compiles the
integration tier).

- **Concurrency (ADR-0015):** a repo-wide grep for `sync.Mutex` / `sync.RWMutex`
  across `internal` and `cmd` returns **zero hits**. Shared state is coordinated
  via `atomic.Value` (the health/discovery snapshot), `atomic` counters, and a
  single-owner channel-fed metrics goroutine; the only `sync` usage is
  `sync.Once` and `sync.WaitGroup`, both off the request path. No `panic(` on the
  request path; errors wrapped with `%w`.
- **Dependency rule (ADR-0003):** `internal/model` imports no internal package;
  `internal/router` and `internal/backend` import only `internal/model` (never
  `internal/server` or each other); `internal/server` imports
  model + router + observability and not backend; `cmd/router` is the sole
  composition root. The server owns its `Router` collaborator interface; the
  `Strategy` interface now has a real polymorphic dispatch site
  (`strategyFor → Execute`).
- **Dependencies (ADR-0010/0015):** stdlib-only except the single sanctioned
  `gopkg.in/yaml.v3`.
- **Testing (ADR-0012):** unit + handler tests are table-driven and stdlib-only;
  `.github/workflows/ci.yml` runs `gofmt`, `go vet`, `go build`, and
  `go test -race ./...`; an integration tier behind `//go:build integration`
  exercises a live router and is excluded from default CI.
- **Behavioral invariants:** transparent passthrough (unknown fields survive
  byte-for-byte; only `model` rewritten, `plugins` stripped), the no-failover
  after first-byte rule, native Anthropic→Anthropic full fidelity (`tools`,
  `tool_choice`, `top_k`, `metadata`, `cache_control` survive), cross-protocol
  field drop, and provenance headers (`X-Router-Model` / `X-Router-Backend` on
  proxy *and* fusion responses) were each confirmed by reading the code and
  running the corresponding tests.
